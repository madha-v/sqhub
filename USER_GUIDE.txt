## 1. Introduction

### What is Mini SQL Compiler?
The Mini SQL Compiler is a complete implementation of a SQL query compiler that demonstrates all phases of compilation:
- **Lexical Analysis**: Breaking down SQL queries into tokens
- **Syntax Analysis**: Validating query structure and building Abstract Syntax Trees
- **Execution**: Running queries on an in-memory database

### Project Purpose
This project was created for the Compiler Design course (6th Semester) to demonstrate:
- How compilers work at each phase
- Token recognition and parsing
- Abstract Syntax Tree construction
- Query execution and optimization

### Features at a Glance

âœ… WHERE clauses with multiple conditions
âœ… Beautiful graphical user interface
âœ… Real-time query execution

## 4. SQL Command Reference

### CREATE TABLE

**Purpose**: Create a new table with specified columns

**Syntax**:
```sql
CREATE TABLE table_name (
    column1 datatype,
    column2 datatype,
    ...
)
```

**Supported Data Types**:
- `INT` - Integer numbers (1, 2, 100, -5)
- `VARCHAR(n)` - Text strings up to n characters
- `FLOAT` - Decimal numbers (3.14, 99.99)
- `DATE` - Date values (format: 'YYYY-MM-DD')

**Examples**:
```sql
-- Simple table
CREATE TABLE users (id INT, name VARCHAR(100))

-- Table with multiple types
CREATE TABLE products (
    product_id INT,
    name VARCHAR(200),
    price FLOAT,
    launch_date DATE
)

-- Employee table
CREATE TABLE employees (
    emp_id INT,
    full_name VARCHAR(150),
    salary FLOAT,
    department VARCHAR(50)
)
```

### INSERT INTO

**Purpose**: Add new records to a table

**Syntax**:
```sql
-- Insert with all columns
INSERT INTO table_name VALUES (value1, value2, ...)

-- Insert with specific columns
INSERT INTO table_name (column1, column2) VALUES (value1, value2)
```

**Rules**:
- String values must be in quotes: 'text'
- Numbers don't need quotes: 123 or 45.67
- Values must match column order
- If specifying columns, only those columns get values

**Examples**:
```sql
-- Insert all columns
INSERT INTO students VALUES (1, 'John Doe', 21, 'A')

-- Insert specific columns
INSERT INTO students (id, name) VALUES (2, 'Jane Smith')

-- Insert multiple records (run separately)
INSERT INTO products VALUES (101, 'Laptop', 999.99, '2024-01-15')
INSERT INTO products VALUES (102, 'Mouse', 29.99, '2024-02-01')
INSERT INTO products VALUES (103, 'Keyboard', 79.99, '2024-02-10')
```

### SELECT

**Purpose**: Query and retrieve data from tables

**Syntax**:
```sql
-- Select all columns
SELECT * FROM table_name

-- Select specific columns
SELECT column1, column2 FROM table_name

-- Select with filter
SELECT * FROM table_name WHERE condition
```

**WHERE Operators**:
- `=` Equal to
- `!=` Not equal to
- `<` Less than
- `>` Greater than
- `<=` Less than or equal to
- `>=` Greater than or equal to

**Logical Operators**:
- `AND` Both conditions must be true
- `OR` Either condition can be true

**Examples**:
```sql
-- Get all records
SELECT * FROM students

-- Get specific columns
SELECT name, grade FROM students

-- Filter by one condition
SELECT * FROM students WHERE age > 20

-- Filter by multiple conditions
SELECT * FROM students WHERE age > 18 AND grade = 'A'

-- OR condition
SELECT * FROM students WHERE grade = 'A' OR grade = 'B'

-- Complex condition
SELECT name, age FROM students WHERE age >= 20 AND (grade = 'A' OR grade = 'B')
```

### UPDATE

**Purpose**: Modify existing records in a table

**Syntax**:
```sql
-- Update one column
UPDATE table_name SET column = value WHERE condition

-- Update multiple columns
UPDATE table_name SET column1 = value1, column2 = value2 WHERE condition
```

**âš ï¸ WARNING**: If you don't use WHERE, ALL records will be updated!

**Examples**:
```sql
-- Update single field
UPDATE students SET grade = 'A+' WHERE id = 1

-- Update multiple fields
UPDATE students SET age = 22, grade = 'B' WHERE name = 'John Doe'

-- Update based on condition
UPDATE products SET price = 899.99 WHERE product_id = 101

-- Update multiple records
UPDATE students SET grade = 'A' WHERE age > 20

-- Update all records (dangerous!)
UPDATE students SET grade = 'Pass'
```

### DELETE

**Purpose**: Remove records from a table

**Syntax**:
```sql
-- Delete with condition
DELETE FROM table_name WHERE condition

-- Delete all records
DELETE FROM table_name
```

**âš ï¸ WARNING**: Be careful with DELETE! Always use WHERE unless you want to delete everything.

**Examples**:
```sql
-- Delete specific record
DELETE FROM students WHERE id = 1

-- Delete based on condition
DELETE FROM students WHERE age < 18

-- Delete multiple conditions
DELETE FROM students WHERE age < 18 OR grade = 'F'

-- Delete all records (use with caution!)
DELETE FROM students
```

### DROP TABLE

**Purpose**: Delete an entire table (structure + data)

**Syntax**:
```sql
DROP TABLE table_name
```

**âš ï¸ WARNING**: This permanently deletes the table! Cannot be undone.

**Examples**:
```sql
DROP TABLE students
DROP TABLE products
DROP TABLE employees
```

## 7. Troubleshooting

### Common Errors and Solutions

#### Error: "Python is not recognized as an internal or external command"

**Problem**: Python is not in system PATH

**Solution**:
1. Reinstall Python
2. During installation, CHECK "Add Python to PATH"
3. Restart Command Prompt after installation

**Alternative Solution**:
1. Find Python installation folder (usually C:\Python3x)
2. Right-click "This PC" â†’ Properties
3. Advanced System Settings â†’ Environment Variables
4. Edit "Path" variable
5. Add Python folder path
6. Click OK and restart

#### Error: "No module named 'tkinter'"

**Problem**: tkinter not installed

**Solution**:
- Reinstall Python
- During installation, ensure "tcl/tk and IDLE" is checked
- tkinter comes with Python by default

#### Error: "SyntaxError: Unexpected token"

**Problem**: SQL syntax error in your query

**Common Causes**:
- Misspelled keywords (SELCT instead of SELECT)
- Missing comma between columns
- Unmatched quotes in strings
- Missing VALUES keyword in INSERT

**Solution**:
- Check spelling of all keywords
- Ensure proper spacing
- Match all quotes: 'text' not 'text
- Compare with sample queries

#### Error: "Table 'xyz' does not exist"

**Problem**: Trying to use a table that wasn't created

**Solution**:
1. Click "ğŸ“Š Show Tables" to see available tables
2. Create the table first with CREATE TABLE
3. Check table name spelling (case-sensitive!)

#### Error: "Number of columns and values don't match"

**Problem**: INSERT statement has wrong number of values

**Solution**:
- Count your table columns
- Ensure you provide exactly that many values
- Or specify column names: INSERT INTO table (col1, col2) VALUES (val1, val2)

#### GUI doesn't show data after SELECT

**Problem**: Query might have failed silently

**Solution**:
1. Check Output Console for error messages
2. Verify table has data: SELECT * FROM table_name
3. Try simpler query first: SELECT * FROM students
4. Check WHERE conditions are correct

#### Application closes immediately after double-clicking BAT file

**Problem**: Python error is occurring

**Solution**:
1. Don't double-click BAT file
2. Open Command Prompt
3. Navigate to folder: cd path\to\mini-sql-compiler
4. Run: python gui.py
5. Read error message that appears
6. Common fixes:
   - Install Python if missing
   - Check file names (gui.py, sql_compiler.py must exist)

## 8. Technical Architecture

### Compiler Phases

#### Phase 1: Lexical Analysis (Scanner)

**File**: sql_compiler.py â†’ Class: Lexer

**Purpose**: Convert SQL query string into tokens

**Process**:
```
Input:  "SELECT name FROM students WHERE age > 20"

Scanning:
- Read character by character
- Identify token boundaries
- Recognize keywords, identifiers, operators

Output: [
    Token(SELECT, "SELECT"),
    Token(IDENTIFIER, "name"),
    Token(FROM, "FROM"),
    Token(IDENTIFIER, "students"),
    Token(WHERE, "WHERE"),
    Token(IDENTIFIER, "age"),
    Token(GREATER_THAN, ">"),
    Token(NUMBER, 20)
]
```

**Token Types Supported**:
- Keywords: SELECT, INSERT, UPDATE, DELETE, CREATE, DROP, FROM, WHERE, VALUES, etc.
- Identifiers: table names, column names
- Literals: strings ('text'), numbers (123, 45.67)
- Operators: =, !=, <, >, <=, >=
- Symbols: (, ), *, ,, ;
- Logical: AND, OR

#### Phase 2: Syntax Analysis (Parser)

**File**: sql_compiler.py â†’ Class: Parser

**Purpose**: Validate syntax and build Abstract Syntax Tree (AST)

**Process**:
```
Input: Token stream from Lexer

Parsing:
- Check token sequence follows SQL grammar
- Build hierarchical tree structure
- Validate query structure

Output: AST Node
Example: SelectNode(
    columns = ['name'],
    table = 'students',
    where = WhereClause(column='age', operator='>', value=20)
)
```

**Grammar Rules Implemented**:
```
CREATE  â†’ CREATE TABLE identifier (column_list)
INSERT  â†’ INSERT INTO identifier [(column_list)] VALUES (value_list)
SELECT  â†’ SELECT column_list FROM identifier [WHERE condition]
UPDATE  â†’ UPDATE identifier SET assignment_list [WHERE condition]
DELETE  â†’ DELETE FROM identifier [WHERE condition]
DROP    â†’ DROP TABLE identifier

column_list â†’ identifier [, identifier]*
value_list  â†’ literal [, literal]*
assignment  â†’ identifier = literal
condition   â†’ comparison [logical_op comparison]*
comparison  â†’ identifier operator literal
operator    â†’ = | != | < | > | <= | >=
logical_op  â†’ AND | OR
```

#### Phase 3: Semantic Analysis

**File**: sql_compiler.py â†’ Throughout Parser and Executor

**Purpose**: Validate semantic correctness

**Checks**:
- Table exists before querying
- Column names are valid
- Data types match (implicit)
- Sufficient values in INSERT
- Assignments reference valid columns

#### Phase 4: Execution

**File**: sql_compiler.py â†’ Classes: QueryExecutor, Database

**Purpose**: Execute the validated query

**Components**:

**Database Manager**:
```python
class Database:
    - tables: Dictionary of table structures
    - create_table(): Add new table
    - insert_row(): Add data
    - select_rows(): Query data with filtering
    - update_rows(): Modify records
    - delete_rows(): Remove records
    - drop_table(): Delete table
```

**Query Executor**:
```python
class QueryExecutor:
    - execute(): Route to appropriate handler
    - _execute_create(): Process CREATE TABLE
    - _execute_insert(): Process INSERT
    - _execute_select(): Process SELECT
    - _execute_update(): Process UPDATE
    - _execute_delete(): Process DELETE
    - _execute_drop(): Process DROP TABLE
```

### Data Structures

#### Table Structure
```python
{
    'table_name': {
        'columns': [
            ('column1', 'INT'),
            ('column2', 'VARCHAR(50)'),
            ('column3', 'FLOAT')
        ],
        'rows': [
            {'column1': 1, 'column2': 'value', 'column3': 3.14},
            {'column1': 2, 'column2': 'value2', 'column3': 2.71},
            ...
        ]
    }
}
```

#### AST Node Types
```python
CreateTableNode(table_name, columns)
InsertNode(table_name, columns, values)
SelectNode(columns, table_name, where_clause)
UpdateNode(table_name, assignments, where_clause)
DeleteNode(table_name, where_clause)
DropTableNode(table_name)
WhereClause(column, operator, value, logical_op, next_condition)
```

## Appendix A: Complete Function Reference

### Lexer Functions
- `tokenize()` - Main tokenization method
- `_read_string()` - Parse string literals
- `_read_number()` - Parse numeric literals
- `_read_identifier()` - Parse identifiers/keywords
- `_read_operator()` - Parse comparison operators
- `_skip_whitespace()` - Skip spaces/tabs/newlines
- `_skip_comment()` - Skip SQL comments (--)

### Parser Functions
- `parse()` - Main parsing method
- `_parse_create()` - Parse CREATE TABLE
- `_parse_insert()` - Parse INSERT INTO
- `_parse_select()` - Parse SELECT
- `_parse_update()` - Parse UPDATE
- `_parse_delete()` - Parse DELETE
- `_parse_drop()` - Parse DROP TABLE
- `_parse_where()` - Parse WHERE clause
- `_expect()` - Expect specific token type

### Database Functions
- `create_table()` - Create new table
- `insert_row()` - Add row to table
- `select_rows()` - Query rows
- `update_rows()` - Modify rows
- `delete_rows()` - Remove rows
- `drop_table()` - Delete table
- `_evaluate_where()` - Evaluate WHERE conditions
